# 手動FTPデプロイ

GitHub Actions を使わずに手動でデプロイする場合の手順。
本番環境への初回アップロードや、ワークフロー外での緊急対応時に使う。

---

## テスト環境へのデプロイ

1. `npm run build` でビルド（`dist/` に出力）
2. FTPクライアント（FileZilla等）で `dist/` の中身をサーバーのドキュメントルートにアップロード

---

## 本番環境へのデプロイ

1. `npm run build` でビルド
2. `dist/` の中身を本番サーバーのドキュメントルートにアップロード

### 本番デプロイ前のチェック

- [ ] テスト環境で表示・動作を確認済み
- [ ] `config/site.config.js` の `baseUrl` が本番URLになっている
- [ ] OGP画像・faviconが差し替え済み

---

## 一部ファイルのみ更新する場合の注意

差分だけをアップロードするときは、ビルド後のファイル名・パスが変わっていないか確認する。

### ハッシュ付きファイル名に注意

Vite はビルドのたびに JS・CSS のファイル名にハッシュを付与する（例: `assets/index-Bx3kA1f2.js`）。
ソースを1行でも変更してビルドすると別名ファイルが生成されるため、**古いファイルを上書きできない**。

- 差分アップロードの場合でも、**`assets/` ディレクトリごと丸ごと差し替える**のが安全。
- HTML も `assets/` 内のファイルを参照しているため、**HTML と `assets/` は必ずセットでアップロードする**。

### 画像・フォントなど静的ファイルのみの変更

JS・CSS・HTML を変更せず、画像やフォントだけ差し替える場合は該当ファイルのみのアップロードで問題ない。
ただし、画像の追加・削除を伴う場合は HTML も再ビルド・再アップロードが必要になる。

---

## アップロード除外ファイル

以下はアップロード不要（`dist/` には含まれないが念のため確認）。

- `.git*`
- `node_modules/`
- `.DS_Store`
- `Thumbs.db`
