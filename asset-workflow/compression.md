# アセット圧縮の考え方

圧縮の目的・判断基準・原則を記録する。実行コマンドは案件側に置く。

---

## 目的

- ページ表示速度の改善
- 通信量の削減（特にモバイル）
- Core Web Vitals（LCP等）の改善

---

## 判断基準

| 状況 | 対応 |
|---|---|
| 画像が100KB以上 | 圧縮を検討する |
| 動画をループ再生する | WebMまたは最適化済みMP4を使う |
| フォントを複数ウェイト使う | 使うウェイトだけサブセット化する |
| Retina対応が必要 | 2x画像 + `srcset` で出し分ける |

---

## 原則

- **元素材は案件側に残す** — 圧縮前のオリジナルは案件リポジトリで管理する。このリポジトリには置かない。
- **実行コマンドは案件側に残す** — `npm run compress` などのスクリプトは案件ごとに定義する。
- **共通repoには「考え方」を残す** — 「なぜ圧縮するか」「何を基準にするか」の判断軸だけここに書く。

---

## フォーマット別の考え方

- **JPEG** — 写真・グラデーション画像。品質80前後が目安。
- **PNG** — 透過が必要な場合のみ使う。不要なら JPEGかWebPに変換。
- **WebP** — JPEG/PNGの代替として積極的に使う。対応ブラウザを確認する。
- **SVG** — アイコン・ロゴは SVGを優先する。不要なメタデータを削除する。
- **動画** — 装飾目的のループ動画は短く・軽く。音声なしで自動再生する。
