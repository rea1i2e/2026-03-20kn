# ルール記述フォーマット

`coding-rules/` にルールを追加するときのフォーマット定義。

---

## フォーマット

```
- **{結論}** — {理由}
  - 例: {具体的な例}
  - 例外: {例外があれば}
```

---

## 各項目の説明

| 項目 | 説明 | 必須 |
|---|---|---|
| 結論 | 「〜する」という能動的な表現で書く | 必須 |
| 理由 | なぜそうするかを1文で書く | 必須 |
| 例 | 具体的なコード・状況・ファイル名など | 任意 |
| 例外 | このルールが適用されないケース | 任意 |

---

## 記述例

```
- **既存実装を観察してから変更する** — 独自スタイルを持ち込むと保守コストが上がるため。
  - 例: 命名規則・ファイル構成・コメントスタイルを先に確認する
  - 例外: 既存実装が明らかに問題を抱えている場合はリファクタを提案する
```

---

## 注意

- 結論は短く（15文字以内が目安）
- 理由は「〜のため」「〜から」で終わる形にする
- 例外は書けるなら書く。なければ省略してよい
