# コーディング規約

## 全体共通

- コミットメッセージ・コードコメントは日本語で記述する
- PHPやEJSファイルで、HTMLタグ内のテキストコンテンツは自動補完しない

### ディレクトリ構成

```
テーマ/
├── assets/          # ビルド成果物（直接編集しない）
├── components/      # 再利用可能なPHPコンポーネント
├── functions-lib/   # 機能別関数ファイル（func-*.php）
├── src/
│   ├── js/          # JSソース（webpackでバンドル）
│   └── sass/        # Sassソース
├── functions.php    # エントリーポイント（functions-lib/を自動読み込み）
└── front-page.php   # トップページテンプレート
```

### ファイル命名規則

| 種別 | 命名パターン | 例 |
|------|-------------|-----|
| PHPコンポーネント | `components/*.php` | `top-mv.php`, `c-button.php` |
| PHP関数ファイル | `functions-lib/func-*.php` | `func-base.php` |
| JSモジュール | `src/js/_*.js` | `_drawer.js` |
| Sassパーツ | `src/sass/components/_p-*.scss` | `_p-header.scss` |
| Sassレイアウト | `src/sass/layout/_l-*.scss` | `_l-header.scss` |
| Sassユーティリティ | `src/sass/utility/_u-*.scss` | `_u-sp.scss` |

## PHP

### 配列記法
`array()` は使用しない。必ず `[]` を使用する。

```php
// ✅
$items = ['key' => 'value'];

// ❌
$items = array('key' => 'value');
```

### データ取得とHTMLの分離
データ取得部分とHTML記述部分は必ず分離する。

```php
<?php
// ✅ データ取得を先に行う
$nav_items = [
  ['slug' => 'about', 'text' => ''],
  ['slug' => 'news',  'text' => ''],
];
?>
<nav class="p-header__pc-nav">
  <?php foreach ($nav_items as $item): ?>
    <a href="<?php page_path($item['slug']); ?>"><?php echo esc_html($item['text']); ?></a>
  <?php endforeach; ?>
</nav>
```

### 関数命名
テーマ固有の関数には `my_` プレフィックスを付ける。

```php
function my_setup() { ... }
add_action('after_setup_theme', 'my_setup');
```

### コンポーネントの読み込み
再利用可能なコンポーネントは `get_template_part()` で読み込む。

```php
<?php get_template_part('components/top-mv'); ?>

<?php get_template_part('components/c-heading', null, [
  'en' => 'News',
  'ja' => '',
]); ?>
```

### コンポーネントファイルの構造
`components/` 内のファイルは、引数を先頭で受け取りHTMLを出力する。

```php
<?php
/**
 * コンポーネント名
 * @param array $args
 *   - key (string): 説明
 */
$value = isset($args['key']) ? $args['key'] : '';
?>
<div class="c-component">
  <?php echo esc_html($value); ?>
</div>
```

### URLヘルパー関数

```php
<img src="<?php img_path('/common/logo.svg'); ?>" alt="" width="" height="">
<a href="<?php page_path('news'); ?>"></a>
<a href="<?php page_path(); ?>"></a>
```

### セキュリティ（出力エスケープ）

| 用途 | 関数 |
|------|------|
| 通常テキスト | `esc_html()` |
| 属性値 | `esc_attr()` |
| URL | `esc_url()` |
| HTML許可 | `wp_kses_post()` |

### エラーハンドリング（早期return）

```php
// ✅
if (!$news_query->have_posts()) return;

// ❌
if ($news_query->have_posts()) { ... }
```

### コメント記法

```php
/**
 * func-xxx
 * ファイルの目的を記述
 */

/*--------------------------------------
  セクション名
--------------------------------------*/
function my_function() { ... }
```

## Sass/SCSS

### クラス名プレフィックス

| プレフィックス | 用途 | 例 |
|--------------|------|-----|
| `p-` | パーツ（ページ固有コンポーネント） | `.p-header`, `.p-top-mv` |
| `l-` | レイアウト | `.l-header`, `.l-inner` |
| `c-` | 汎用UIコンポーネント | `.c-button`, `.c-heading` |
| `u-` | ユーティリティ | `.u-sp`, `.u-pc` |
| `js-` | JS操作対象（IDに使用） | `#js-drawer` |

### BEM記法（フラット記述）
入れ子（ネスト）は `mq()` 以外では使用しない。すべてフラットに記述する。

```scss
// ✅
.p-header { background-color: #ccc; }
.p-header.is-show { background-color: #eee; }
.p-header__inner { display: flex; }
.p-header__logo { width: rem(40); }

// ❌ ネストは使わない
.p-header {
  &__inner { display: flex; }
  &__logo { width: rem(40); }
}
```

### サイズ指定
サイズは原則 `rem()` 関数を使用する。`font-size` は `maxrem()` を使用する（最小10px保証）。

```scss
.p-header__inner { padding-inline: rem($padding-sp); }
.p-header__logo { width: rem(40); }
.p-header__title { font-size: maxrem(14); }
```

### メディアクエリ
`mq()` ミックスインを使用する。デフォルトはスマホファースト（`md: 768px` 以上）。

```scss
.p-header__menu-button {
  @include mq() { display: none; }
}
.p-header__pc-nav {
  display: none;
  @include mq() { display: block; }
}
```

### @use構文
`@import` は使用しない。`@use` を使用する。

```scss
// コンポーネントファイルの先頭
@use "global" as *;
```

### グローバル関数・変数（`src/sass/global/`）

| ファイル | 内容 |
|---------|------|
| `_setting.scss` | デザインカンプ幅、インナー幅、余白変数 |
| `_function.scss` | `rem()`, `maxrem()`, `vwsp()`, `vwpc()`, `maxvwpc()` 等 |
| `_breakpoints.scss` | `mq()` ミックスイン、ブレークポイント定義 |

## JavaScript

### モジュール構成
ES6モジュール形式（`import`/`export`）を使用する。モジュールファイルはアンダースコア始まり。

```javascript
// index.js（エントリーポイント）
import './_drawer.js';
import './_slider.js';
```

### 要素の存在確認（早期return）

```javascript
// ✅
const el = document.getElementById('js-top-mv-splide');
if (!el) return;
new Splide(el, { type: 'fade' }).mount();

// ❌
if (el) { new Splide(el, { type: 'fade' }).mount(); }
```

### アクセシビリティ
インタラクティブ要素の状態変化時は `aria-*` 属性を更新する。

```javascript
menuButton.setAttribute('aria-expanded', isOpen ? 'true' : 'false');
drawer.setAttribute('aria-hidden', isOpen ? 'false' : 'true');
```
