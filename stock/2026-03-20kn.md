## 全体共通

- コミットメッセージ・コードコメントは日本語で記述する
- PHPやEJSファイルで、HTMLタグ内のテキストコンテンツは自動補完しない
- データ取得とHTML記述は可能な限り分離する

## PHP

### 配列記法
`array()` は使用しない。必ず `[]` を使用する。

```php
// ✅
$items = ['key' => 'value'];

// ❌
$items = array('key' => 'value');
```

### データ取得とHTMLの分離
データ取得部分とHTML記述部分は必ず分離する。

```php
<?php
// ✅ データ取得を先に行う
$nav_items = [
  ['slug' => 'about', 'text' => ''],
  ['slug' => 'news',  'text' => ''],
];
?>
<nav>
  <?php foreach ($nav_items as $item): ?>
    <a href="<?php page_path($item['slug']); ?>"><?php echo esc_html($item['text']); ?></a>
  <?php endforeach; ?>
</nav>
```

### エラーハンドリング（早期return）

```php
// ✅
if (!$query->have_posts()) return;

// ❌
if ($query->have_posts()) { ... }
```

## EJS / HTML

### データ分離の実装方針
- `<% ... %>` ブロックで必要なデータ（配列/オブジェクト等）を準備し、描画はその後に書く
- HTML側では `<%- ... %>` / `<%= ... %>` で参照するだけにする

NG（避ける）：
- HTMLタグ内に新規の文言をそのまま書き足す
- `<% ... %>` とHTML描画が細かく交互に混在する（ロジック過多）

OK（例の型）：
- `const pageIntro = { ... };` を用意して `include` に渡す

## Sass/SCSS

### クラス名プレフィックス

| プレフィックス | 用途 | 例 |
|--------------|------|-----|
| `p-` | パーツ（ページ固有コンポーネント） | `.p-header`, `.p-top-mv` |
| `l-` | レイアウト | `.l-header`, `.l-inner` |
| `c-` | 汎用UIコンポーネント | `.c-button`, `.c-heading` |
| `u-` | ユーティリティ | `.u-sp`, `.u-pc` |
| `js-` | JS操作対象（IDに使用） | `#js-drawer` |

### BEM記法（フラット記述）
入れ子（ネスト）は `mq()` 以外では使用しない。すべてフラットに記述する。

```scss
// ✅
.p-header { background-color: #ccc; }
.p-header__inner { display: flex; }

// ❌ ネストは使わない
.p-header {
  &__inner { display: flex; }
}
```

### サイズ指定
- サイズは原則 `rem()` 関数を使用する
- `font-size` は `maxrem()` を使用する（最小10px保証）

### メディアクエリ
`mq()` ミックスインを使用する。モバイルファースト方針。

```scss
.p-header__pc-nav {
  display: none;
  @include mq() { display: block; }
}
```

### @use構文
`@import` は使用しない。`@use` を使用する。

```scss
@use "global" as *;
```

### idセレクタ
SCSSでは `id` セレクタを使用しない。

## JavaScript

### モジュール構成
ES6モジュール形式（`import`/`export`）を使用する。モジュールファイルはアンダースコア始まり。

```javascript
import './_drawer.js';
import './_slider.js';
```

### 要素の存在確認（早期return）

```javascript
// ✅
const el = document.getElementById('js-top-mv');
if (!el) return;

// ❌
if (el) { ... }
```

### セレクタ運用
- JSで操作する要素は `data-*` 属性または `js-` 接頭辞の `id` / `class` で管理する
- 既存コードでは `id="js-..."` が多用されている（例：`js-header`, `js-drawer`）

### アクセシビリティ
インタラクティブ要素の状態変化時は `aria-*` 属性を更新する。

```javascript
menuButton.setAttribute('aria-expanded', isOpen ? 'true' : 'false');
drawer.setAttribute('aria-hidden', isOpen ? 'false' : 'true');
```
