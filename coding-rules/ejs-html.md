# EJS / HTML

---

## テキストコンテンツの自動補完禁止

- **HTMLタグ内のテキストは自動補完しない** — デザイン・要件にある文言のみを使用する。
  - 例: 見出し・段落・ボタン文字は設計の文言（または既存実装）から参照する
  - 例外: altテキストなど意味的に空でよい場合は `alt=""` のまま残す

---

## データとHTMLの分離

- **データを先に組み立て、HTMLで参照するだけにする** — ロジックと描画が混在すると読みにくく、保守コストが上がるため。
  - 例: `<% ... %>` ブロックでオブジェクト/配列を準備し、`<%- ... %>` / `<%= ... %>` で出力する

```ejs
<%
const items = [
  { label: '', href: '/about/' },
  { label: '', href: '/news/' },
];
%>
<ul>
  <% items.forEach(item => { %>
    <li><a href="<%= item.href %>"><%- item.label %></a></li>
  <% }); %>
</ul>
```

NG（避ける）：
- HTMLタグ内に新規の文言をそのまま書き足す
- `<% ... %>` とHTML描画が細かく交互に混在する（ロジック過多）

---

## includeの書き方

- **includeパスは設定ファイルが提供する変数（`ejsPath` 等）を使う** — パスのハードコードを避け、設定変更に強くするため。
  - 例: `include(ejsPath + 'common/_head.ejs', { page })`

---

## 出力演算子の使い分け

- **HTMLエスケープが必要な値は `<%= ... %>`、HTMLをそのまま出力する場合は `<%- ... %>` を使う** — XSSリスクを避けるため、用途を明確に区別する。
  - 例: テキスト値 → `<%= page.title %>`
  - 例: HTML文字列 → `<%- page.bodyHtml %>`
