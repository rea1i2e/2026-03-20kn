# アセット圧縮の考え方

圧縮の目的・判断基準・原則を記録する。実行コマンドは案件側に置く。

---

## 目的

- ページ表示速度の改善
- 通信量の削減（特にモバイル）
- Core Web Vitals（LCP等）の改善

---

## 判断基準

| 状況 | 対応 |
|---|---|
| 画像が100KB以上 | 圧縮を検討する |
| 動画をループ再生する | WebMまたは最適化済みMP4を使う |
| フォントを複数ウェイト使う | 使うウェイトだけサブセット化する |
| Retina対応が必要 | 2x画像（表示サイズの2倍解像度）を基本とする |

---

## 原則

- **元素材は案件側に残す** — 圧縮前のオリジナルは案件リポジトリで管理する。このリポジトリには置かない。
- **実行コマンドは案件側に残す** — `npm run compress` などのスクリプトは案件ごとに定義する。
- **共通repoには「考え方」を残す** — 「なぜ圧縮するか」「何を基準にするか」の判断軸だけここに書く。

---

## 制作環境別の圧縮方針

### 静的サイト（Vite）

- **圧縮は `vite build` の中で自動実行する** — `@vheemstra/vite-plugin-imagemin` により JPEG/PNG/GIF/SVG を圧縮する。
- **WebP・AVIF への変換は `site.config.js` の `imageAltFormats` で制御する** — `'webp'` / `'avif'` / `'both'` / 無効を案件ごとに選択する。
- **圧縮後も 1MB を超える場合は手動圧縮を実施する** — Squoosh 等で個別に対応する。

### WordPress（Vite + プラグイン）

- **WebP・AVIF への変換は WordPress プラグインに委ねる** — Vite 側では変換を行わない（`VITE_ENABLE_WEBP=false` がデフォルト）。
- **Vite では JPEG/PNG/GIF/SVG の圧縮のみ実施する** — 品質は環境変数（`VITE_JPEG_QUALITY` 等）で調整できる。
- **フォーマット変換プラグインの選定は案件ごとに判断する** — EWWW Image Optimizer 等を使う。

---

## フォーマット別の考え方

- **JPEG** — 写真・グラデーション画像。品質 75〜85 が目安（ファーストビューは 80、サムネイルは 75）。
- **PNG** — 透過が必要、またはイラスト・図解など色数が少なくエッジがくっきりしている画像に使う。それ以外は WebP か JPEG に変換。圧縮レベルは 8〜9（可逆圧縮のため最大圧縮でよい）。
- **WebP** — JPEG/PNG の代替として積極的に使う。品質 75〜80 が目安。
- **SVG** — アイコン・ロゴは SVG を優先する。不要なメタデータを削除する。
- **動画** — 詳細は [video.md](video.md) を参照。

---

## フォーマット優先順位

**SVG > WebP > JPEG**（透過が不要なら PNG は使わない）

### 静的サイト（Vite）

AVIF + WebP へのフォールバックを基本とする。`after-build.mjs` が `<picture>` タグへの変換と `<source>` の挿入を自動で行う。変換するフォーマットは `site.config.js` の `imageAltFormats` で制御する。

### WordPress

JPEG / PNG を基本とし、WebP・AVIF への変換はプラグインに委ねる。プラグインがサーバー側で変換・配信を行うため、HTML 側では通常の `<img>` タグで記述する。

---

## 用途別のサイズ目標

| 用途 | 目標サイズ | 備考 |
|---|---|---|
| ファーストビュー画像 | 200KB 以下 | LCP に直結するため優先的に削減する |
| サムネイル・繰り返し画像 | 50KB 以下 | 複数枚表示されるため1枚あたりを小さくする |
| OGP 画像 | 300KB 以下 | 推奨サイズ 1200×630px |

---

## 読み込み優先度・遅延読み込み

- **`loading="lazy"` はファーストビュー外の画像に付ける** — ファーストビュー内（LCP 対象候補）には付けない。付けると初期表示が遅くなる。
- **LCP 対象画像には `fetchpriority="high"` を付ける** — ブラウザの優先度ヒューリスティックを上書きして早期取得させる。
- **`width` / `height` 属性は自動付与されるため手書き不要** — 静的サイトは `after-build.mjs`（sharp で取得）、WordPress は `func-images.php`（`theme-assets.json` または `getimagesize` で取得）が自動で付与する。手動で書いた場合も上書きされる。
