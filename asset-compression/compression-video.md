# 動画の考え方

動画の圧縮・フォーマット・設置の判断基準を記録する。実行コマンドは案件側に置く。
HTML / PHP での読み込み方は [coding-rules/video.md](../coding-rules/video.md) を参照。

---

## 基本方針

- **圧縮・変換には ffmpeg を使う** — 案件側にコマンドを定義して実行する。
- **元素材は案件側に残す** — 圧縮前のオリジナルは案件リポジトリで管理する。

---

## フォーマットの考え方

- **MP4（H.264）** — 互換性最優先。コンテンツ動画のデフォルト。全ブラウザで再生できる。
- **WebM（VP9）** — MP4 より軽量になるケースが多い。ループ背景など軽量化を優先する場面で MP4 と併用する。

---

## 用途別の方針

### ループ背景・装飾動画・ヒーロー動画

- できるだけ短くする（目安: 10秒以内。ただし「10秒」に強い根拠はなく、実質的な制約はサイズ目標の 3MB 以下。3MB に収まるなら秒数は問わない）。
- 解像度は表示サイズに合わせて下げる（フル HD 不要なら 720p 以下）。
- WebM と MP4 の両方を用意する。

### コンテンツ動画（説明・紹介）

- **外部サービス（YouTube 等）への埋め込みを先に検討する** — 自己ホストより配信品質・負荷の面で有利なことが多い。
- 自己ホストする場合は MP4（H.264）を基本とする。

---

## サイズ目標

| 用途 | 目標サイズ | 備考 |
|---|---|---|
| ループ背景・ヒーロー動画 | 3MB 以下 | モバイルでの通信量を考慮する |
| コンテンツ動画（自己ホスト） | — | できるだけ外部サービスに委ねる |

---

## git 管理の判断基準

### 基本方針

- **動画（圧縮済み・元素材）は git 管理しない。 `.gitignore` で除外する。**
- **理由：バイナリは差分管理できず、リポジトリが大きくなるため。**

### .gitignore に追加する拡張子

```gitignore
# 動画ファイル
*.mp4
*.webm
*.mov
*.avi
```

### 例外（管理してよいケース）

| ケース | 判断 |
|--------|------|
| 数十KB以下のごく小さい装飾用動画 | 管理してもよい（目安: 1MB 未満） |
| テスト・CI 用のサンプル動画 | 専用ディレクトリに限定して管理してもよい |

### 動画の受け渡し・保管

- **圧縮前の元素材はクライアントから受け取ったまま案件フォルダ（git 管理外）に保管する**
- **圧縮済みファイルはサーバーへ直接アップロードするか、別途ファイル共有で納品する**
- **ffmpeg コマンドは `package.json` の `scripts` に残す** — 再圧縮が必要になったときに再現できるようにするため。
